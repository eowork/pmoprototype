SECTION 1: ARTIFACT GOVERNANCE OVERVIEW
================================================================================

The PMO Dashboard project follows the ACE (Analyze, Contextualize, Execute)
framework with TWO living documents:

  1. research.md  - Research findings and technical analysis
  2. plan.md      - Execution contract with actionable steps

All other research files are archived in docs/archive/ with date-based naming.

--------------------------------------------------------------------------------
CORE PRINCIPLES:
--------------------------------------------------------------------------------

  1. TWO LIVING DOCUMENTS ONLY
     - No creating multiple research_*.md files
     - Consolidate findings into research.md
     - Archive historical content, don't proliferate

  2. SINGLE SOURCE OF TRUTH
     - No duplicate information across files
     - Each fact lives in exactly one place
     - Cross-reference by section number

  3. COMPACTION OVER PROLIFERATION
     - Completed work goes into <details> collapsed sections
     - Active work stays visible
     - Archive when context window exceeds 200 lines per section

  4. 60-SECOND RULE
     - Reader must identify next action within 60 seconds
     - plan.md THE NOW section answers "What do I do next?"

SECTION 5: ANTI-PATTERNS TO AVOID
================================================================================

DO NOT:
  x Create multiple research_*.md files for different topics
  x Append new phases without compacting completed ones
  x Mix execution tasks with historical logs in plan.md
  x Use version numbers in living document filenames (e.g., plan_v2.md)
  x Duplicate information that exists in research.md into plan.md

DO:
  + Consolidate findings into single research.md
  + Archive historical content with date-based naming
  + Keep plan.md THE NOW section to 1-4 items maximum
  + Use section references instead of copying content
  + Compact completed work into <details> sections
